## 新增需求

### 需求：生成分析简报
系统必须能够根据上传的易问单 Excel 文件和用户指定的关注点，生成结构化的业务分析简报。

#### 场景：成功生成简报
- **当** 用户上传有效的易问单 Excel 文件（`.xls` 或 `.xlsx`）并输入“分析关注点”后提交
- **那么** 系统应解析 Excel 数据，统计工单总量、Top 分类及关键词
- **那么** 系统应调用 Dify 工作流生成文本报告
- **那么** 系统应保存报告记录并返回生成的报告内容和统计摘要

#### 场景：文件格式错误
- **当** 用户上传非 Excel 格式的文件（如 `.txt`, `.pdf`）
- **那么** 系统应拒绝请求并返回“文件格式不支持”的错误提示

#### 场景：数据解析失败
- **当** 用户上传的 Excel 文件缺少必要的列（如“自分类”、“一线人员身份”）
- **那么** 系统应返回“文件内容缺失关键列”的错误提示

### 需求：查看历史简报
系统必须提供查询历史分析简报的接口，支持分页查询。

#### 场景：分页查询历史
- **当** 用户请求获取简报列表（包含页码和每页数量）
- **那么** 系统应返回按创建时间倒序排列的简报列表数据（包含ID、文件名、关注点、创建时间）

### 需求：查看简报详情
系统必须允许用户通过 ID 查看特定简报的详细内容。

#### 场景：查看详情
- **当** 用户请求查看指定 ID 的简报详情
- **那么** 系统应返回该简报的完整内容，包括分析报告文本和统计摘要数据

### 需求：导出分析简报
系统必须支持将生成的分析简报导出为 Markdown 或文本文件。

#### 场景：导出报告
- **当** 用户点击“导出”按钮
- **那么** 系统应触发下载，文件内容为当前的分析报告文本
