# 提议：网信安涉诈信息核查与数据隔离功能

**Author**: Gemini Agent
**Date**: 2026-02-04
**Status**: Proposed

## 1. 摘要 (Summary)
本提议旨在完善网信安涉诈信息管理模块，通过引入基于用户属地的数据隔离机制，并新增独立的“涉诈信息核查”前端界面，实现分公司与总公司在涉诈线索处理上的职责分离。
总公司负责全量监控与录入，分公司负责处理分配到本属地的核查任务。

## 2. 目标 (Goals)
*   **数据安全与隔离**：实现后端数据访问控制，确保非超级管理员用户仅能查看和操作其所属地市（部门）的数据。
*   **业务流程分离**：通过新增独立的“涉诈信息核查”菜单，为分公司核查人员提供专注的工作视图，避免全量数据干扰。
*   **精准补录**：提供专用的核查补录接口，限制仅能修改 9 个指定的核查类字段，防止误改核心线索数据。

## 3. 需求分析 (Requirements)
### 3.1 用户角色
*   **超级管理员/总公司用户**：拥有全局视野，可查看所有属地数据，拥有所有菜单权限。
*   **分公司用户**：仅拥有局部视野，仅能查看 `join_location` 匹配其部门名称的数据，主要职责是进行核查补录。

### 3.2 功能需求
*   **列表查询**：自动根据登录用户身份过滤数据。
*   **核查补录**：支持对单条线索进行补充调查信息的录入。
*   **界面分离**：新增“涉诈信息核查”页面，与现有的“涉诈信息管理”页面并列。

## 4. 字段定义 (Fields)
本次涉及的补录字段（倒数 9 项）：
1.  `is_compliant` (是否合规受理)
2.  `has_resume_before` (涉诈涉案前是否有复通)
3.  `is_resume_compliant` (复通是否规范)
4.  `responsibility` (责任认定)
5.  `is_self_or_family` (是否本人或亲属涉诈涉案)
6.  `police_collab` (警企协同情况)
7.  `investigation_note` (调查户主备注)
8.  `abnormal_scene` (异常场景识别)
9.  `feedback` (核查情况反馈)
