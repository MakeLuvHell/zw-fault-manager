# 提议：网信安模块功能增强与数据库分表优化

**Author**: Gemini Agent
**Date**: 2026-02-05
**Status**: Implemented

## 1. 摘要 (Summary)
本次变更旨在提升“网信安涉诈信息核查”页面的用户体验与系统安全性，并对数据库结构进行分表重构，以解决内外系统并发写入时可能产生的数据覆盖风险。

## 2. 功能增强 (Features)
*   **权限与接口分离**：拆分管理岗与核查岗的列表接口，管理岗可见全量数据，核查岗受属地过滤。
*   **交互优化**：核查页面引入“待核查”与“已核查”Tab 分页，支持实时数量统计角标。
*   **搜索增强**：新增“入网属地”和“涉诈月份”搜索项，且月份搜索通过时间戳范围匹配实现格式兼容。
*   **全量导出**：新增 Excel 导出功能，覆盖全量 36 个字段。

## 3. 架构优化 (Architecture)
*   **分表重构**：将原本的大宽表 `t_wx_safe_info` 拆分为：
    *   `t_wx_safe_info` (主表)：记录线索核心信息与核查结论，由本系统维护。
    *   `t_wx_safe_detail` (附表)：记录业务画像、话单等扩展信息，供外部程序更新。
*   **冗余与过滤**：在主表保留 `join_location` 以确保过滤性能，在附表冗余 `phone_number` 方便外部程序检索。
*   **自动展平**：后端通过 Pydantic Validator 自动展平关联数据，前端无需感知后端表结构变化。
