# 提议：添加网信安涉诈信息管理模块

**Author**: Gemini Agent
**Date**: 2026-02-04

## 1. 摘要 (Summary)
本提议旨在新增“网信安涉诈信息管理”模块，主要解决涉诈线索信息的录入与管理问题。
系统将支持单条数据录入和 Excel 批量导入，并对核心字段（线索编号、业务号码）进行严格校验。

## 2. 目标 (Goals)
*   实现专用的数据表 `t_wx_safe_info`，包含完整的 36 个业务字段。
*   实现单条数据录入功能，支持线索编号唯一性校验和 11 位业务号码校验。
*   实现 Excel 批量导入功能，提供详细的失败反馈（含失败总数、线索编号及原因）。
*   前端实现基础的查询与录入界面，初期仅展示核心 8 字段。

## 3. 数据结构 (Schema Design)
**Table**: `t_wx_safe_info`
**Schema**: `wxsafe`

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `clue_number` | String(100) | 线索编号 (PK) |
| `phone_number` | String(20) | 业务号码 (11位) |
| ... | ... | (完整字段见 Spec) |
