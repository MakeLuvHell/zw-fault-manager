## 新增需求
### 需求：自动外呼任务调度
系统必须包含一个后台定时任务，用于自动处理外呼数据的筛选与推送。

#### 场景：定时触发
- **当** 系统时间达到预设的调度时间点（默认每5分钟）
- **那么** 系统应自动触发“自动外呼任务”

#### 场景：数据去重筛选
- **当** 任务开始执行
- **那么** 系统应读取 `call_task` 表的全量数据
- **且** 系统应读取 `call_history` 表的历史数据
- **且** 系统应筛选出所有 `mobile_phone` 不在历史记录中的任务作为“新增任务”

#### 场景：API 推送与重试
- **当** 存在新增任务时
- **那么** 系统应依次对每个任务调用第三方推送 API
- **且** 请求体应包含唯一的 `distinct_id`（格式：`YYYYMMDDHHmmss` + 3位序列号）
- **且** 如果 API 调用失败，系统应自动重试 3 次
- **且** 每次请求之间应保持配置的间隔时间

#### 场景：历史记录更新
- **当** 所有新增任务处理完成（无论成功失败）
- **那么** 系统应将**本次推送成功**的记录更新到 `call_history` 表
- **且** 更新操作应先清空旧历史，再写入新记录（保持原子性）
