## 新增需求
### 需求：SQL 校验与解析
系统必须提供 SQL 校验功能，以确保查询语句的合法性并辅助用户配置。

#### 场景：校验带参数的 SQL
- **当** 用户提交包含绑定参数的 SQL（例如 `SELECT * FROM t WHERE id = :id`）
- **那么** 系统不应报错
- **且** 系统应返回 SQL 的输出列结构（Columns）
- **且** 系统应识别并返回 SQL 中的输入参数名（Params，例如 `["id"]`）

#### 场景：参数名智能提示
- **当** 用户在配置节点参数名时
- **那么** 系统应提供下拉选项
- **且** 选项内容应为该 SQL 中实际使用的绑定参数名
