## 新增需求
### 需求：外呼任务配置管理
系统必须提供一个可视化界面，用于管理外呼任务的配置，包括数据源、字段映射和调度设置。

#### 场景：查看任务列表
- **当** 用户访问外呼任务管理页面
- **那么** 系统应展示任务数据表格，包含任务名称、Cron 表达式、源数据表、启用状态和备注
- **且** 表格支持分页和刷新

#### 场景：新增外呼任务
- **当** 用户点击"新增任务"按钮
- **那么** 系统应弹出任务编辑弹窗
- **且** 表单包含基础配置、数据源配置、业务参数映射三个板块
- **且** 基础配置包含：任务名称、Cron 表达式、状态开关
- **且** 数据源配置包含：schema.table_name 格式的源数据表选择器
- **且** 业务参数映射包含：手机号字段、员工姓名字段、系统名称字段、工单类型字段、工单数量字段的列名映射

#### 场景：编辑外呼任务
- **当** 用户点击任务行的"编辑"按钮
- **那么** 系统应弹出预填充的任务编辑弹窗
- **且** 用户可修改任务配置后保存

#### 场景：删除外呼任务
- **当** 用户点击任务行的"删除"按钮或勾选多行后点击"批量删除"
- **那么** 系统应弹出确认对话框
- **且** 确认后删除选中的任务

#### 场景：立即执行任务
- **当** 用户点击任务行的"立即执行"按钮
- **那么** 系统应异步触发该任务的外呼处理
- **且** 前端应立即显示"任务已触发"的反馈，不阻塞界面

### 需求：数据源元数据查询
系统必须支持获取数据库元数据，包括跨 Schema 的表和字段信息。

#### 场景：获取 Schema 列表
- **当** 用户打开数据源配置区域
- **那么** 系统应加载并展示所有可用的数据库 Schema

#### 场景：获取表字段列表
- **当** 用户选择源数据表后
- **那么** 系统应自动加载该表的所有字段供字段映射选择

### 需求：字段映射自动绑定
系统必须自动将用户选择的字段映射到 API 参数。

#### 场景：手机号字段自动绑定
- **当** 用户选择"手机号字段"的数据库列名
- **那么** 系统应自动将该值同时绑定到 API 的 `accs_nbr` 和 `contact_nbr` 参数

#### 场景：员工姓名字段自动绑定
- **当** 用户选择"员工姓名字段"的数据库列名
- **那么** 系统应自动将该值同时绑定到 API 的 `cust_name` 和 `staff_name` 参数
