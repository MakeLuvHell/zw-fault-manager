## 新增需求

### 需求：工单 Excel 上传与校验
系统必须允许用户上传 `.xlsx` 格式的工单报表，并验证是否包含必要的列（标题、时长、类型、处理人、处理时间）。

#### 场景：上传合法的工单报表
- **当** 用户选择一个包含正确列名的 Excel 文件并点击上传
- **那么** 系统必须成功解析文件内容并将其转换为内部 JSON 格式

#### 场景：上传格式错误的报表
- **当** 用户上传的文件缺少关键列（如“处理时长”）
- **那么** 系统必须返回明确的错误提示并拒绝后续 AI 分析

### 需求：Dify AI 简报生成
系统必须将解析后的 JSON 数据发送至 Dify 工作流，并根据用户输入的“分析关注点”生成 Markdown 格式的简报。

#### 场景：触发 AI 分析
- **当** 后端收到合法的解析数据和分析关注点（如“关注处理效率”）
- **那么** 系统必须异步调用 Dify API，并将返回的 Markdown 内容与元数据关联存储

### 需求：报告历史查询与展示
系统必须提供一个列表页面，允许用户查看过去生成的分析报告，并在详情页以美观的 Markdown 格式渲染内容。

#### 场景：查看报告详情
- **当** 用户在列表中点击某个分析报告
- **那么** 系统必须从数据库读取存储的 Markdown 文本，并使用前端渲染引擎完整呈现分析结果
