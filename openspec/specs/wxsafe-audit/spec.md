# 网信安涉诈信息审计规范 (WxSafe Audit Spec)

## 1. 概述
本模块负责网信安涉诈信息的全生命周期审计，提供操作日志的记录、存储和查询服务，以满足合规性要求和数据追溯需求。

## 2. 需求

### 需求：操作日志自动记录
系统必须在每次对“涉诈信息”进行更新（Update）操作时，自动记录操作日志。

#### 场景：更新涉诈信息触发日志记录
- **当** 用户通过 API 或界面更新某条线索的任何字段（如修改反馈内容、责任认定等）
- **那么** 系统必须在 `wxsafe_fz_log` 表中创建一条新记录
- **并且** 记录必须包含操作人 ID、操作类型（UPDATE）、操作时间
- **并且** 记录必须包含变更前的完整快照（before_data）和变更后的完整快照（after_data）

### 需求：操作日志查询
系统必须提供 API 以查询指定线索编号的所有历史变更记录，并按时间倒序排列。

#### 场景：查询线索变更历史
- **当** 用户请求查看线索 `DX20250901001` 的历史轨迹
- **那么** 系统返回该线索的所有日志记录列表
- **并且** 列表按操作时间从新到旧排序
- **并且** 每条记录应包含操作人姓名、操作时间、变更类型及变更详情
